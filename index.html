Concurrency Programming

What is concurrency?

"Concurrency is running several tasks at same time."

"Concurrency is when a task is broken down into smaller pieces and run simutaneously."

One way to achieve conncurency is with multithreading. and for multithreading we need threads.

Threads

A thread is an independent path of code execution.

Share process-wide memory and resources.

Cannot exist outside of the process lifetime.

Have their own stack and memory management.

memory management means each thread have their own local variables

Processes vs Threads

Every process has at least one main thread.

A process can have multiple threads.

Threads from the same process can interact with each other without using IPC mechanisms.

Non sincronized thread execution can produce unpredictable results.

Inter Process Communications ex: sockets, signal handlers or semaphores.

Thread Creation

There two ways to create threads:

  • Extending the Thread class
  • Implement the Runnable interface
Second way is preferred, better when using anonymous class

Thread Synchronization

To avoid unpredictable results we need to control thread execution.

Locks are the proper way to achieve thread-sync.

Other valid approach is to use Thread .join() method.

Avoid loops and using Thread .sleep() method.

We will learn about locks later
